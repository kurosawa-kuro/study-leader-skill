# 受験要件と学習戦略

## 1. この章で切れる選択肢

- すぐ問題演習に入る
- 受験要件を先に満たす
- PM 専業レベルまで深掘りする
- 合格点突破型で進める

## 2. 技術の最短定義

CAPM は、プロジェクト管理の基礎知識を問う知識試験である。
この戦略では、技術リードに必要な PM 管理観点を外部証明するために使う。

## 3. 今の状態で選ぶ条件

- 予算、人月、スコープ、リスク管理の説明力が弱い
- リモート環境での推進再現性を補強したい
- PM 専業化ではなく、管理観点の基礎を体系化したい

## 4. 今の状態で選ばない条件

- 取得自体が目的化している
- PMLE や技術学習を大きく圧迫する
- 受験要件を確認せず見切り発車する

## 5. ひっかけパターン

- 学習時間を必要以上に膨らませる
- 用語集としてしか学ばない
- 23 時間教育要件を軽視して、申込直前に詰まる

## 6. 代表パターン

| 項目 | 内容 |
|---|---|
| 実技 | なし |
| 問題数 | 150 問 |
| 試験時間 | 180 分 |
| 受験方法 | Pearson VUE 会場またはオンライン監督 |
| 日本語対応 | あり |
| 受験要件 | 高卒相当以上 + 23 時間以上の PM 教育 |

## 7. 実務へのつなぎ方

- 「受験資格を満たした」だけでなく、「何を補うために取るか」を言う
- 23 時間講座は修了証が残るものを選ぶ
- 学習順は、受験要件確認 → 講座受講 → 分野学習 → 受験判断にする

## 8. 最後に覚える一文

CAPM は、技術リードが PM 管理観点を体系化して示すための基礎証明である。
