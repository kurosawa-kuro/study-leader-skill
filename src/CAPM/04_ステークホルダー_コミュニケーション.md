# ステークホルダー・コミュニケーション

## 1. この章で切れる選択肢

- ステークホルダー管理
- コミュニケーション管理
- 報告不足のまま自己完結する
- 全員に同じ粒度で伝える

## 2. 技術の最短定義

ステークホルダー管理は、関係者ごとに期待値と必要情報を整理することを指す。
コミュニケーション管理は、何を、誰に、いつ、どの粒度で伝えるかを設計することを指す。

## 3. 今の状態で選ぶ条件

- リモートで作業過程が見えにくい
- 顧客、派遣先、開発者で必要情報が違う
- 高評価を再現可能な説明力に変えたい

## 4. 今の状態で選ばない条件

- 報告を単なる進捗読み上げにする
- 全員に同じ詳細を投げる
- 論点整理せずに会議へ出る

## 5. ひっかけパターン

- 伝えた量が多ければ十分と思う
- 問題報告を遅らせる
- 技術的に正しければ説明不要と思う

## 6. 代表パターン

| 相手 | 期待されること | 伝えるべき内容 |
|---|---|---|
| 顧客 | 判断材料 | 影響、選択肢、優先順位 |
| 派遣先 / 上位 | 安定運用 | 進捗、論点、リスク |
| 開発メンバー | 実行条件 | 方針、役割、依存関係 |

## 7. 実務へのつなぎ方

- 報告は「やったこと」だけでなく、「次に何を判断してほしいか」まで含める
- リモートでは、論点整理と早めのリスク共有が評価に直結する
- CAPM の学習内容は、会議運営ではなく期待値調整の説明力として回収する

## 8. 最後に覚える一文

コミュニケーション管理は、情報を流すことではなく、相手が判断できる形で渡すことである。
